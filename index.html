HOME MATE

부동산 추천 AI 컨시어지 서비스

2026.02 - 2026.03 (2개월)

팀 프로젝트

01. 한줄 요약

사용자의 자연어 대화와 라이프스타일을 분석하여 최적의 부동산 매물을 찾아주는 AI 컨시어지 서비스

02. 배경 및 목표

기존 부동산 앱의 복잡한 필터 검색 방식에 피로도를 느끼는 사용자들에게, 일상적인 대화만으로도 원하는 조건(예: "강남역 도보 10분, 월세 100 이하")과 주변 인프라까지 고려한 맞춤형 매물을 추천하는 혁신적인 UX를 제공하고자 기획했습니다.

03. 역할 및 기여도
  • 팀장 (기획 총괄 및 백엔드/AI 리드)
  • 대규모 데이터 수집 파이프라인 구축 및 MongoDB 스키마 설계
  • AI 챗봇 엔진 및 검색 백엔드 로직 100% 구현
04. 사용 기술 및 도구
Python Flask MongoDB Atlas LangChain OpenAI API Web Crawling Map API
05. 진행 과정
  • 크롤링 및 공공데이터를 결합해 약 3만 건의 매물과 4만 건의 인프라 데이터를 직접 수집·전처리하여 DB 적재 및 공간 매핑.
  • LangChain과 GPT 프롬프트를 활용, 자연어를 MongoDB 검색용 JSON 필터로 자동 변환하는 파이프라인 구축.
  • 추천 정확도를 높이기 위해 '다중 조건 가중합 기반의 스코어링 모델' 설계 및 데이터 튜닝 진행.
06. 결과 및 성과
  • UI 조작 없이 텍스트 대화만으로 복합 조건 매물을 검색하는 서비스 상용화 달성.
  • 공간 인덱싱 최적화를 통해 평균 데이터 조회 시간을 0.2초 이내로 단축 (80% 이상 성능 향상)시켰습니다.
07. 어려움 및 해결 과정

문제: 수만 건의 방대한 공간 데이터를 다중 조건으로 검색할 때 심각한 서버 부하와 속도 지연(병목) 발생.

해결: MongoDB Atlas로 전환 후 논리적 스키마 재설계. $geoWithin, $centerSphere 공간 연산 쿼리 및 공간 인덱스를 적극 도입하여 검색 속도를 획기적으로 개선.

08. 배운 점

물리적인 제약이 적은 NoSQL(MongoDB) 환경일수록, 서비스 성능을 극대화하기 위해서는 애플리케이션 레벨에서의 정교한 데이터베이스 스키마 설계와 인덱싱 최적화가 필수적이라는 것을 뼈저리게 체득했습니다.

Mise

유튜브 영상 기반 요리 레시피 추출 및 최저가 장보기 서비스

2026.01 - 2026.02 (2개월)

팀 프로젝트

01. 한줄 요약

유튜브 요리 영상에서 레시피를 자동 추출하고 이커머스와 연동해 1인분 최저가 장보기 리스트를 제공하는 시스템

02. 배경 및 목표

유튜브 요리 영상을 보고 직접 요리하려는 사용자들이 레시피를 따로 메모하고 식재료를 쇼핑몰에서 일일이 검색해야 하는 번거로움을 해결하기 위해, 영상 시청부터 최저가 장보기까지 한 번에 이어지는 원스톱(One-stop) 서비스를 목표로 했습니다.

03. 역할 및 기여도
  • 데이터 추출 파이프라인 및 백엔드 로직 설계 리드
  • 비동기 처리(Asyncio) 아키텍처 도입
  • 데이터 규격화 알고리즘 구현
04. 사용 기술 및 도구
Python (Asyncio) LangChain OpenAI API (Whisper) Tavily Search API
05. 진행 과정
  • OpenAI Whisper(STT)를 활용해 영상 음성을 텍스트로 추출하고, Tavily Search API를 연동한 동적 검색 로직 구현.
  • 비표준화된 수량 표현('약간', '1큰술')을 표준 규격(g, ml)으로 매핑하는 알고리즘을 개발하여 최저 단가 계산의 정확도 향상.
  • 실제 구매 페이지 URL을 추출하여 바로 이동할 수 있는 링크 연동 기능 개발.
06. 결과 및 성과
  • 수동 검색 과정을 완전히 생략하고 구매 전환율을 높일 수 있는 동적 자동화 로직 완성.
  • 비동기 병렬 처리를 통해 기존 2분 소요되던 파이프라인 응답 시간을 30초로 단축 (75% 성능 최적화)했습니다.
07. 어려움 및 해결 과정

문제: 여러 이커머스 쇼핑몰 API를 순차 호출(직렬 처리) 시, 네트워크 응답을 기다리는 I/O 병목 현상으로 대기 시간이 2분 이상 소요됨.

해결: 파이썬의 Asyncio를 도입하여 비동기 병렬 처리 구조로 전면 개편. 복수의 검색 요청을 동시에 실행하고 취합하여 네트워크 대기 시간을 최소화함.

08. 배운 점

외부 API 호출이 잦은 I/O Bound 작업에서는 단순한 로직 개선보다, 비동기 프로그래밍(Asyncio)과 같은 아키텍처적인 접근이 성능 최적화에 압도적으로 강력하다는 것을 실전 경험을 통해 배웠습니다.
